# Development Guide

## Git Flow

main

↓

develop

↓

feature/*

Never push directly to main.

---

## Branch Naming

feature/dashboard

feature/interview-ui

feature/mood-engine

bugfix/pdf

hotfix/login

---

## Commit Style

feat:

fix:

refactor:

style:

docs:

---

## Folder Structure

frontend/

backend/

docs/

assets/

---

## Coding Rules

Keep components reusable.

Keep functions small.

Avoid duplicated logic.

Comment only when necessary.

Never hardcode secrets.

---

## Code Review

Every PR should verify

✓ Responsive

✓ Error handled

✓ Loading state

✓ Accessibility

✓ Mobile tested

---

## Definition of Done

Feature is complete only if

✓ Works

✓ Responsive

✓ Tested

✓ Error handled

✓ Loading state

✓ Styled

✓ Reviewed
